⚠ Known Limitations
🛒 Amazon
🚫
Region-restricted products — Items listed as "not available in your country" return no data. The page itself shows a restriction wall, not product info.
⏳
CAPTCHA frequency — On cloud/datacenter IPs, Amazon challenges almost every request. Residential proxy strongly recommended for production deployment.
💰
Complex price ranges — Products with "From $X" or configurable pricing (custom engraving, bundle packs) may return the base price only, not variant-specific pricing.
🔄
Variant-to-price mapping — While all variant options are extracted, the price per specific variant combination (e.g. Red + XL = $24.99) requires a separate API call per combination. Currently returns the base price.
🖤 Zara
🛡
Akamai CDN blocks — Zara uses Akamai WAF which fingerprints browser properties. Datacenter IPs are consistently blocked. Home/residential IPs work reliably. Chrome headless-new mode is enabled to reduce fingerprinting.
🌍
Region-specific URLs — A UK Zara URL must be tested from a UK-localized browser context. Cross-region testing may give "Access Denied".
📷
Image quality — If XHR API is not captured, HTML fallback images may be lower resolution (placeholder lazy-load sources). This is rare but possible.
🛍 SHEIN
☁
Cloudflare challenge — SHEIN uses Cloudflare Bot Management. Datacenter/VPS IPs are blocked with a JS challenge page. Residential proxy required for server deployments.
🔄
Changing API endpoints — SHEIN updates their internal API URL patterns frequently. The interceptor uses broad keyword matching to handle this, but very new URL patterns may be missed.
🌐
Regional storefronts — us.shein.com, in.shein.com, uk.shein.com have slightly different API structures. The parser handles all known variants but a new storefront may need testing.
🚀 Improvement Plan (Phase 2)
HIGH
Residential Proxy Integration Add proxy rotation (e.g. Bright Data, Oxylabs, or Smartproxy) for production server deployments. This single change will raise success rates to 90%+ on all 3 sites. ~$50–100/month for a starter plan.
HIGH
Variant-to-Price Mapping (Amazon) For products where each variant has a different price: make a secondary AJAX call to Amazon's price API per variant combination. Return `variant_prices: { "Red-XL": "24.99", "Blue-XL": "26.99" }` in the response.
MED
Add H&M, ASOS, Nike These are the next most-requested sites. Each needs a new scraper file (~1–2 days per site). Framework is ready — just wire up new scrapers using the documented template.
MED
Response Caching (Redis) Cache extraction results by URL for 15 minutes. Same product URL requested twice in a short window returns instantly. Reduces CAPTCHA exposure and response time for repeated lookups.
MED
SHEIN Cloudflare Bypass Integrate FlareSolverr (open-source) as a Layer 0 before Playwright. FlareSolverr handles the JS challenge automatically, significantly improving SHEIN success rate on server IPs.
LOW
Async Extraction (Parallel Layers) Run HTTP and Playwright layers concurrently — return whichever finishes first successfully. Cuts Amazon's worst-case time from 30s to ~10s.
LOW
Webhook / Callback Mode For slow Playwright paths, return a job ID immediately and POST the result to a callback URL when done. Prevents app UI from blocking during 15–25s extractions.
